# 90분 커리큘럼

## 학습 목표

교육 종료 시점에 임원진이 다음을 할 수 있어야 한다.

1. **설명**: 바이브 코딩 / 에이전틱 코딩 / 하네스 엔지니어링의 차이를 본인의 언어로 설명한다.
2. **판단**: 본인 업무에서 Claude Code를 쓸 장면 1개 이상을 구체적으로 지목한다.
3. **지시**: 팀에 "우리 CLAUDE.md를 만들어봅시다" 수준의 구체적 지시를 내릴 수 있다.

## 핵심 경험

**"내가 알던 AI 도구가 조직 설계의 레버가 될 수 있다"** 를 직접 목격하는 것.

## 설계 근거

| 이론 | 적용 |
|---|---|
| Andragogy (Knowles) | 첫 5분에 relevance 선언 |
| Cognitive Load (Sweller) | worked example 위주, 실습은 1회 짧게, 외재적 부하 제거 |
| TAM (Davis) | perceived usefulness가 임원 본인에게 형성되도록 설계 |
| Executive Ed | 체험 → 성찰 → 적용 계획 사이클 |
| 3-tier 프레임 | Vibe → Agentic → Harness |

## 타임테이블

| 시간 | 섹션 | 분 | 모드 | Tier | 성공 신호 |
|---|---|---|---|---|---|
| 0~10 | Opening: 3-tier 프레임 | 10 | 강의 + 녹화 데모 | 전체 | "내 역할이 무엇인지 궁금해짐" |
| 10~25 | T1 체험 — 문서 분석 L1~L2 | 15 | Worked Example | T1 | "어? 파일을 알아서 열어봤네" |
| 25~40 | T2 이해 — 문서 분석 L3~L5 | 15 | Worked Example | T2 | "이건 ChatGPT와는 다르다" |
| 40~55 | 실습 — 본인 노트북에서 L2 재현 | 15 | 핸즈온 | T1→T2 | 본인 손으로 성공 경험 |
| 55~75 | **T3 하네스 엔지니어링** | 20 | 강의 + 라이브 데모 | **T3** | "이게 내 레버다" |
| 75~85 | 도입 시 고려사항 | 10 | 강의 | 조직 | 보안·품질 질문이 발생 |
| 85~90 | Action Planning | 5 | 워크시트 | — | 월요일 액션 1개 작성 |

## 섹션 상세

### 0~10분 · Opening

**목표**: 90분의 프레임 선언 + 기대치 재설정

1. (1분) 도착점 선언: "오늘 여러분의 도착점은 T3 하네스 엔지니어링입니다. 지금은 낯선 단어지만, 90분 후에는 본인의 언어로 설명할 수 있습니다."
2. (2분) **녹화 데모 재생**: 가벼운 웹앱 한 방 생성 (vibe coding 샘플)
3. (1분) 재프레이밍: "지금 보신 것은 T1입니다. 흥미롭지만 여러분 본인이 월요일에 할 일은 아닙니다."
4. (6분) 3-tier 프레임 설명
   - **T1 Vibe coding**: 즉흥적 한 방 프롬프트
   - **T2 Agentic coding**: 계획·실행·검증 루프
   - **T3 Harness engineering**: 조직의 AI 사용 방식을 설계

### 10~25분 · T1 체험 (L1~L2)

대상: [`sample-team/`](../sample-team/) 가상 팀 문서 폴더

- **L1**: "`sample-team/회의록/2026-03-15-주간회의.md` 뭐라고 써있어?" — 단일 파일 요약
- **L2**: "`sample-team/` 폴더 전체를 보고 이 팀이 뭐 하는 팀인지 설명해줘" — 자동 탐색

**성공 신호**: 수강자가 L2에서 Claude가 여러 파일을 스스로 열어보는 장면에 반응한다.

### 25~40분 · T2 이해 (L3~L5)

- **L3**: "최근 회의록 3개를 읽고 아직 처리 안 된 액션 아이템만 담당자별로 정리해줘"
- **L4**: "다음 주 임원회의 아젠다를 계획으로 만들어줘. 근거 문서 링크 포함"
- **L5**: "그 계획대로 `sample-team/회의록/2026-04-12-임원회의-아젠다.md` 파일을 실제로 만들어줘"

**성공 신호**: L4에서 "계획" 이 나오는 순간 임원진이 "작업 쪼개기를 AI가 한다고?" 반응을 보인다.

### 40~55분 · 실습

- 각자 노트북에서 `sample-team/` 폴더에 L2 수준 질문 1개 던지기
- 성공 보장 프롬프트(핸드아웃)를 제공한다
- 실습 직후 30초 페어 회고: "방금 무엇이 놀라웠나요?"

**성공 보장 프롬프트**:
> sample-team 폴더 전체를 훑어보고 이 팀의 Q2 주요 과제 3개를 우선순위와 함께 알려줘.

### 55~75분 · T3 하네스 엔지니어링

**핵심 메시지**: "여러분은 코드를 짤 필요가 없습니다. 여러분은 **규칙을 설계**합니다."

1. **(10분) CLAUDE.md 라이브 데모**
   - 빈 CLAUDE.md 상태에서 Claude에게 "다음 주 임원회의 아젠다를 만들어줘" 요청 → 결과 관찰
   - CLAUDE.md에 "모든 응답은 존댓말로 작성한다", "회의 아젠다는 최대 5개 항목까지만 쓴다" 두 줄 추가
   - 동일 요청 반복 → 동일 Claude의 행동 변화를 실시간으로 관찰
   - 프레이밍: "이 한 줄이 조직 전체의 AI 동작 기준이 됩니다."

2. **(5분) Skills 라이브 데모**
   - 반복되는 워크플로우를 이름으로 부른다는 개념
   - 이 교육 자료 제작에 쓴 `expert-panel` skill 호출 장면을 재생
   - 프레이밍: "반복되는 의사결정을 이름으로 만드는 것. 조직 지식의 코드화."

3. **(3분) Agents 언급만**
   - "역할 기반 위임" 개념만 소개
   - 예: 코드 리뷰어, 보안 검토자 에이전트
   - 라이브 데모는 생략

4. **(2분) Hooks 언급만**
   - "정책의 자동 집행" 개념만 소개
   - 예: 커밋 전 정책 검증, 민감 정보 차단
   - 라이브 데모는 생략

### 75~85분 · 도입 시 고려사항

- **보안**: 민감 데이터 경계, 사내 리포지터리 취급
- **품질**: 검증 없이 AI 결과 수용 금지, 사람의 리뷰 루프
- **팀 온보딩**: CLAUDE.md를 팀 문서의 일부로 관리
- **거버넌스**: 라이선스, 감사 추적

### 85~90분 · Action Planning

워크시트 1장을 작성한다. 세부는 [action-worksheet.md](action-worksheet.md) 참조.

1. 월요일 오전, 내가 Claude Code로 시도할 일 1개
2. 팀에 요청할 CLAUDE.md 초안 주제 1개
3. 다음 1개월 안에 지켜볼 것 1개
