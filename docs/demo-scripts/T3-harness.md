# 하네스 엔지니어링 라이브 데모 대본

20분 섹션. 세션 2의 agentic 루프가 **어떤 환경·규칙 안에서** 돌지를 설계하는 구간이다.

## 핵심 메시지

> **Agent = Model + Harness.** agentic 루프는 강력하지만, 그 루프가 도는 환경을 설계하지 않으면 결과는 일정하지 않다. 하네스는 모델을 제외한 모든 것(도구·컨텍스트·가드레일·피드백·관측)이다.

출처:
- LangChain, [The Anatomy of an Agent Harness](https://blog.langchain.com/the-anatomy-of-an-agent-harness/)
- Martin Fowler / Böckeler, [Harness Engineering](https://martinfowler.com/articles/exploring-gen-ai/harness-engineering.html) — **Guides(피드포워드)** 와 **Sensors(피드백)** 설계 실천

---

## 1. CLAUDE.md 라이브 데모 (10분)

### 목적

조직의 AI 동작 기준을 **한 파일**로 바꿀 수 있다는 것을 실시간으로 목격하게 한다.

### 준비

세션 시작 전 `CLAUDE.md`를 최소 상태로 되돌려둔다.

```markdown
# CLAUDE.md

이 저장소는 Claude Code 90분 입문 교육 자료입니다.
```

### 장면 1 — 빈 상태에서 요청 (3분)

**프롬프트**:
```
다음 주 회의 아젠다 초안을 3줄로 만들어줘. sample-team 근거 문서는 안 봐도 돼.
```

Claude는 일반적인 톤으로 아젠다를 생성한다. 형식이 고정되지 않는다.

**강사 멘트**: "일반적인 결과입니다. 우리의 규칙은 아무것도 반영되지 않았습니다."

### 장면 2 — CLAUDE.md에 세 줄 추가 (3분)

수강자가 보는 화면에서 직접 타이핑한다.

```markdown
## 조직 규칙

- 모든 응답은 존댓말로 작성한다.
- 회의 아젠다는 최대 5개 항목까지만 작성한다.
- 각 아젠다 항목은 한 줄을 넘지 않는다.
```

**강사 멘트**: "지금 타이핑한 이 세 줄이 오늘부터 이 저장소의 AI 동작 기준입니다."

### 장면 3 — 동일 요청 반복 (3분)

**동일한 프롬프트**를 다시 친다.

```
다음 주 회의 아젠다 초안을 3줄로 만들어줘. sample-team 근거 문서는 안 봐도 돼.
```

Claude가 존댓말·항목 수·한 줄 제약을 지킨 결과를 출력한다. **이 차이가 수강자 눈앞에서 일어나는 것이 핵심**이다.

**강사 멘트**: "같은 요청입니다. 차이는 단 하나 — CLAUDE.md에 세 줄을 적은 것뿐입니다. 이것이 하네스의 첫 번째 요소입니다."

### 장면 4 — 응용 아이디어 1분

- "모든 SQL 쿼리는 쓰기 전에 팀장 리뷰를 받는다"
- "고객 데이터가 포함된 분석은 반드시 익명화부터 한다"
- "한국어 문서는 `~합니다` 체를 쓴다"
- "커밋 메시지는 [JIRA-번호]로 시작한다"

## 2. Skills 라이브 데모 (5분)

### 목적

반복되는 워크플로우를 **이름으로 부르는** 개념을 이해시킨다.

### 장면 1 — 개념 설명 (2분)

**강사 멘트**: "CLAUDE.md가 '항상 적용되는 규칙'이라면, Skills는 '필요할 때 부르는 이름 붙은 절차'입니다. 예를 들어 '코드 리뷰', '고객 이슈 분류', '회의록 정리' 같은 반복 작업을 이름으로 만들어두는 것입니다."

### 장면 2 — 실제 예시 (3분)

강사는 **이 교육 자료를 만들 때 실제로 사용한** skill을 보여준다.

**강사 멘트**: "이 커리큘럼을 설계할 때 저는 `전문가소환` 이라는 skill을 썼습니다. '전문가 6명을 불러서 근거 기반으로 토론해줘'라는 워크플로우에 이름을 붙여둔 거죠. 한 번 만들어두면 다음부터는 이름만 부르면 됩니다."

---

## 3. Agents 언급 (3분)

**개념만 소개. 라이브 데모는 생략.**

**강사 멘트**: "세 번째 요소는 Agents입니다. **역할 기반 위임**입니다."

- "코드 리뷰어 에이전트" — PR 품질·보안 검토
- "보안 검토자 에이전트" — 민감 정보 포함 여부 확인
- "회의록 요약 에이전트" — 긴 회의록을 3줄로

"각 에이전트는 **서로 다른 CLAUDE.md와 다른 Skills를 가질 수 있습니다.** 역할별로 다른 규칙을 따릅니다."

---

## 4. Hooks 언급 (2분)

**개념만 소개. 라이브 데모는 생략.**

**강사 멘트**: "마지막 요소는 Hooks입니다. **정책의 자동 집행**입니다."

- "커밋 전에 민감 정보 스캔" — 자동 차단
- "배포 전에 테스트 통과 여부 확인" — 자동 차단
- "고객 데이터 접근 시 로그 기록" — 자동 실행

"CLAUDE.md는 '지켜주세요'고 Hooks는 '못 하게 막는다'입니다. **강제력 있는 규칙**이 필요할 때 Hooks를 씁니다."

## 섹션 마무리 1분

"오늘의 한 줄 — **Agent = Model + Harness.** 모델을 뺀 나머지 전부가 하네스이고, 그 하네스는 CLAUDE.md, Skills, Agents, Hooks 네 가지 재료로 만들 수 있습니다."
