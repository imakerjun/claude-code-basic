# L1~L5 문서 분석 데모 스크립트

가상 팀 문서 폴더([`sample-team/`](../../sample-team/))를 대상으로 5단계 그라데이션 데모를 진행한다.

## 공통 준비

- Claude Code 작업 디렉터리: `claude-code-basic/` 루트
- 강사는 세션 전 L1~L5 전체를 최소 1회 처음부터 끝까지 실행한다.
- 화면은 2분할: 왼쪽 Claude Code 터미널, 오른쪽 `sample-team/` 파일 탐색기 또는 에디터.

---

## L1 — 단일 파일 요약

**목적**: 기존 대화형 AI와의 유사점에서 출발. 의도적으로 낮게 시작.

**프롬프트**:
```
sample-team/회의록/2026-03-15-주간회의.md 뭐라고 써있어?
```

**예상 반응**: "음, ChatGPT와 비슷하네."

**강사 멘트**: "여기까지는 이미 익숙한 AI 도구와 비슷합니다. 한 단계 올라가겠습니다."

---

## L2 — 폴더 탐색과 종합

**목적**: Claude가 스스로 여러 파일을 열어본다는 것을 목격하게 한다. Anthropic 정의의 *"agentic coding environment"* 가 실제로 드러나는 첫 장면.

**프롬프트**:
```
sample-team/ 폴더 전체를 훑어보고 이 팀이 어떤 팀이고 무엇을 하는지 설명해줘.
```

**관찰 포인트**: Claude가 Glob, Read를 여러 번 호출하는 도구 사용 로그가 터미널에 표시된다. 이 장면을 **손가락으로 가리키며** 강조한다.

**강사 멘트**: "Claude가 파일을 하나씩 열어봤습니다. 아무도 '이 파일을 열어봐라'고 지시하지 않았습니다. 이것이 agentic 루프의 첫 증거입니다."

---

## L3 — 여러 문서 가로질러 종합·판단

**목적**: 매주 하고 싶지만 수작업이면 번거로운 일 — "여러 회의록 가로질러 미처리 액션만 뽑기" — 이 루프 안에서 어떻게 한 번에 처리되는지 보여준다.

**프롬프트**:
```
sample-team/회의록/ 폴더의 회의록 3개를 모두 읽고,
아직 처리되지 않은 액션 아이템만 담당자별로 정리해줘.
완료된 것은 제외해줘.
```

**예상 출력 형태**:
```
미처리 액션 아이템 (담당자별)

이대리
- [2026-03-29] 본부 대시보드 KR1/KR2 지표 스키마 협의 (4/3까지)

최매니저
- [2026-03-29] severity 2%/5분/shadow 정의서 v2 배포 (4/5까지)

정개발
- [2026-03-29] 온보딩 자동화 레거시 파악 (4/5까지)
```

**강사 멘트**: "다문서를 가로질러 판단하는 작업입니다. 사람이 하면 수 분, 이 루프에서는 수 초. 중요한 건 '시간 절약'이 아니라 '루프가 어떻게 움직이는가'입니다."

---

## L4 — 다단계 계획 수립

**목적**: "계획을 먼저 세워달라"는 요청이 agentic 루프의 전형적 형태임을 보여준다.

**프롬프트**:
```
다음 주 회의 아젠다를 만들려고 해.
sample-team/ 문서들을 근거로 아젠다 계획을 먼저 세워줘.
- 우선순위가 높은 항목부터
- 각 항목마다 근거 문서를 링크로 달아줘
- 아직 파일을 만들지는 말고, 계획만 보여줘
```

**관찰 포인트**: Claude가 TodoWrite 또는 단계별 plan을 제시한다.

**강사 멘트**: "이전까지는 '물어보고 답을 받는' 관계였습니다. 지금부터 Claude는 **작업을 쪼개고, 순서를 정하고, 그 순서대로 스스로 실행**합니다. Anthropic이 정의한 agentic coding environment의 모습입니다."

---

## L5 — 실행 루프 (상태 변경)

**목적**: 읽기만 하던 루프가 **상태를 바꾸는 루프**로 넘어가는 지점을 보여준다.

**프롬프트**:
```
좋아. 그 계획대로 sample-team/회의록/2026-04-12-회의-아젠다.md 파일을 실제로 만들어줘.
```

**관찰 포인트**:
- Claude가 Write 도구를 호출
- 파일이 생성되는 즉시 파일 탐색기에 뜬다
- 에디터에서 열어 내용 확인

**강사 멘트**: "한 줄 지시 — 근거 문서 읽기 → 우선순위 판단 → 계획 수립 → 파일 생성. 이 4단계가 한 번의 요청에 연결되는 것이 agentic 루프의 정의입니다."

---

## 마무리 질문

"여기까지가 agentic 루프의 모습입니다. 한 가지 의문이 생깁니다 — **'이 Claude가 어떤 기준으로 우선순위를 정했지?'** 그 기준을 어떻게 조직이 통제할 수 있을까요? 그게 하네스 엔지니어링입니다."

→ 하네스 세션으로 자연스러운 브리지.

---

## 예상 실패 시나리오

| 상황 | 대응 |
|---|---|
| L3에서 Claude가 완료된 항목도 포함 | "완료된 것 제외해줘" 한 번 더 요청. 이것도 루프의 일부. |
| L5에서 파일 생성 권한 확인 프롬프트 | 수락. "Claude는 민감한 작업 전에 사람 확인을 받습니다" 라고 자연스럽게 안내. |
| 출력이 너무 길어 스크롤로 가려짐 | "길죠. 다음 세션에서 이걸 짧게 만드는 방법을 보여드립니다." → 하네스 CLAUDE.md로 자연스러운 브리지. |
