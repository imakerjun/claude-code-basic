# L1~L5 문서 분석 데모 스크립트

가상 팀 문서 폴더([`sample-team/`](../../sample-team/))를 대상으로 5단계 그라데이션 데모를 진행한다.

## 공통 준비

- Claude Code 작업 디렉터리: `claude-code-basic/` 루트
- 강사는 세션 전 L1~L5 전체를 최소 1회 처음부터 끝까지 실행한다.
- 화면은 2분할: 왼쪽 Claude Code 터미널, 오른쪽 `sample-team/` 파일 탐색기 또는 에디터.

---

## L1 — 단일 파일 요약

**목적**: "ChatGPT와 비슷하네" 진입점. 의도적으로 낮게 시작.

**프롬프트**:
```
sample-team/회의록/2026-03-15-주간회의.md 뭐라고 써있어?
```

**예상 반응**: 임원진 — "음, 그래서?"

**강사 멘트**: "여기까지는 여러분이 이미 아시는 AI 도구와 비슷합니다. 한 단계 올라가겠습니다."

---

## L2 — 폴더 탐색과 종합

**목적**: Claude가 스스로 여러 파일을 열어본다는 것을 목격하게 한다. 첫 차별화 포인트.

**프롬프트**:
```
sample-team/ 폴더 전체를 훑어보고 이 팀이 어떤 팀이고 무엇을 하는지 설명해줘.
```

**관찰 포인트**: Claude가 Glob, Read를 여러 번 호출하는 도구 사용 로그가 터미널에 표시된다. 이 장면을 **손가락으로 가리키며** 강조한다.

**강사 멘트**: "보이시나요? Claude가 파일을 하나씩 열어봤습니다. 아무도 '이 파일을 열어봐라'고 지시하지 않았습니다."

**예상 반응**: "어?"

---

## L3 — 여러 문서 가로질러 종합·판단 (결정적 "아하" 포인트)

**목적**: 임원진이 실제로 매주 하고 싶지만 못 하는 일 — "여러 회의록 가로질러 미처리 액션만 뽑기".

**프롬프트**:
```
sample-team/회의록/ 폴더의 회의록 3개를 모두 읽고,
아직 처리되지 않은 액션 아이템만 담당자별로 정리해줘.
완료된 것은 제외해줘.
```

**예상 출력 형태**:
```
미처리 액션 아이템 (담당자별)

이대리
- [2026-03-29] 신규 입사자 온보딩 가이드 공유 (4/1까지)

최매니저
- [2026-03-29] severity P0 기준 재검토 후 재공유 (4/5까지)

정개발
- [2026-03-29] 온보딩 자동화 현황 파악 (4/5까지)
```

**강사 멘트**: "이것이 매주 30분 걸리던 일입니다. 방금 30초에 끝났습니다. 그리고 이 작업, **여러분이 임원 회의 들어가기 직전에 하고 싶으셨던 거 아닌가요?**"

**예상 반응**: 노트 필기 시작. 이게 아하 순간이다.

---

## L4 — 다단계 계획 수립 (Vibe → Agentic 경계)

**목적**: "계획을 먼저 세워달라" 는 요청이 **에이전틱 코딩의 본질**임을 보여준다.

**프롬프트**:
```
다음 주 임원회의 아젠다를 만들려고 해.
sample-team/ 문서들을 근거로 아젠다 계획을 먼저 세워줘.
- 우선순위가 높은 항목부터
- 각 항목마다 근거 문서를 링크로 달아줘
- 아직 파일을 만들지는 말고, 계획만 보여줘
```

**관찰 포인트**: Claude가 TodoWrite 또는 단계별 plan을 제시한다. "1단계 ... 2단계 ..." 형태.

**강사 멘트**: "여기서 전환점입니다. 이전까지는 '물어보고 답을 받는' 관계였습니다. 지금부터는 Claude가 **작업을 쪼개고, 순서를 정하고, 그 순서대로 스스로 실행**합니다. 이것이 에이전틱 코딩입니다."

---

## L5 — 실행 루프

**목적**: 계획을 실제 파일로 변환하는 자기 수정 루프를 보여준다.

**프롬프트**:
```
좋아. 그 계획대로 sample-team/회의록/2026-04-12-임원회의-아젠다.md 파일을 실제로 만들어줘.
```

**관찰 포인트**:
- Claude가 Write 도구를 호출
- 파일이 생성되는 즉시 파일 탐색기에 뜬다
- 에디터에서 열어 내용 확인

**강사 멘트**: "방금 일어난 일을 정리합니다. 여러분은 '아젠다 만들어줘'라고 한 줄 말했습니다. Claude는 **근거 문서를 읽고, 우선순위를 판단하고, 계획을 세우고, 파일을 만들었습니다.** 이게 T2 에이전틱 코딩입니다."

---

## 마무리 질문

"여기까지 오늘 보신 것이 T2입니다. 그런데 한 가지 의문이 생기시지 않나요? **'이 Claude가 어떤 기준으로 우선순위를 정했지?'** 그 기준을 조직이 어떻게 통제할 수 있을까요? 그게 T3 하네스 엔지니어링입니다. 잠시 후에 이어갑니다."

→ T3 섹션으로 자연스러운 브리지.

---

## 예상 실패 시나리오

| 상황 | 대응 |
|---|---|
| L3에서 Claude가 완료된 항목도 포함 | "완료된 것 제외해줘" 한 번 더 요청. 이것도 T2 대화의 일부. |
| L5에서 파일 생성 권한 확인 프롬프트 | 수락. "Claude는 민감한 작업 전에 사람 확인을 받습니다" 라고 자연스럽게 안내. |
| 출력이 너무 길어 스크롤로 가려짐 | "길죠. 다음 세션에서 이걸 짧게 만드는 방법을 보여드립니다." → T3 CLAUDE.md로 자연스러운 브리지. |
