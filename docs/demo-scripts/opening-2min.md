# 오프닝 2분 웹앱 데모 녹화 대본

## 목적

- 세션 초반 2분 안에 "AI가 코드도 짠다" 임팩트 한 방을 보여준다.
- 이어서 강사가 "이건 Vibe coding입니다" 라고 재프레이밍한다.
- 라이브가 아니라 **녹화 재생**으로 한다. 리스크 제거가 최우선.

## 녹화 시나리오 (단일 HTML, 빌드 없음)

**소재**: 회의 타이머 웹 페이지 (단일 HTML 파일, 의존성 없음)

### 장면 1 — 요청 (10초)

Claude Code 터미널에서 다음 프롬프트를 친다.

```
회의 타이머 웹 페이지를 timer.html 하나로 만들어줘.
- 중앙에 큰 타이머 (MM:SS)
- 시작/일시정지/리셋 버튼
- 배경은 차분한 미드나잇 블루
- 의존성 없이 단일 파일로
```

### 장면 2 — 생성 (40초)

- Claude가 파일을 생성하는 장면을 화면 녹화
- 생성된 `timer.html` 파일을 즉시 브라우저로 열어 동작 확인
- 버튼 클릭, 타이머 동작 장면 포함

### 장면 3 — 개선 1턴 (40초)

다음 프롬프트를 친다.

```
5분·10분·15분 프리셋 버튼을 추가해줘. 디자인 톤은 유지.
```

Claude가 `timer.html`을 수정하는 장면 녹화 → 브라우저 새로고침 → 프리셋 버튼 동작 확인.

### 장면 4 — 마무리 (30초)

화면 캡처: 최종 결과물. 타이머가 돌아가는 화면을 5초 정지.

## 녹화 스펙

- 길이: 2분 내외 (최대 2분 20초)
- 해상도: 1920x1080
- 음성 없음 (현장에서 강사가 내레이션)
- 포맷: mp4

## 현장 내레이션 (강사가 재생 중 말할 것)

1. "Claude Code에 자연어로 요청합니다."
2. "파일이 하나 생성됩니다. 바로 동작합니다."
3. "이어서 한 번 더 자연어로 고쳐달라고 합니다."
4. "여기까지입니다. 이것이 바이브 코딩(Vibe coding)입니다."
5. 재프레이밍: **"오늘 우리의 주제는 이게 아닙니다. 이건 시작점입니다."**

## 주의

- 화려함보다 **단순함**을 선택한다. 복잡한 프레임워크는 쓰지 않는다.
- 녹화본에 민감 정보가 없는지 확인한다.
- 재생 중 문제가 생겨도 당황하지 않는다. 강사의 한 마디 재프레이밍이 본론이다.
