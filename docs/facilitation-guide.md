# 퍼실리테이션 가이드

90분 세션을 강사 입장에서 운영하기 위한 체크리스트와 주의사항.

## 사전 준비 (세션 1일 전까지)

- [ ] `sample-team/` 폴더가 최신 상태인지 확인
- [ ] Claude Code가 강사 노트북에 설치·로그인되어 있음
- [ ] L1~L5 데모 프롬프트를 1회 처음부터 끝까지 리허설 (프롬프트는 `docs/demo-scripts/L1-L5-analysis.md`)
- [ ] T3 CLAUDE.md 데모 리허설 (빈 상태 → 추가 → 차이 관찰)
- [ ] 오프닝 2분 녹화 영상 재생 확인
- [ ] 실습용 성공 보장 프롬프트 핸드아웃 인쇄 또는 슬라이드 준비
- [ ] Action Planning 워크시트 인쇄 (인원수 + 예비 3장)
- [ ] 네트워크·프로젝터·화면 공유 테스트

## 참가자 사전 안내 (세션 3일 전)

- [ ] 노트북 지참 안내
- [ ] Claude Code 설치 안내 ([site reference](../site/content/reference/))
- [ ] 로그인 완료 요청
- [ ] 사전 학습 자료 없음 명시 — 입문 교육임을 강조

## 세션 중 운영 원칙

### 공통

- 질문이 나오면 **그 자리에서 Claude Code로 실험**해서 보여준다. 이 자체가 T2 에이전틱 루프의 증거가 된다.
- "잘 모르겠다"는 답을 두려워하지 않는다. "지금 같이 물어봅시다" 가 임원진에게 가장 강력한 메시지다.
- 매 섹션 끝에 **30초만** "방금 무엇이 놀라웠는가?" 질문을 던진다. 답변을 강요하지 않는다.

### 시간 관리

- T2 섹션(25~40분)이 가장 늦어지기 쉽다. L5 데모는 길어야 5분.
- T3 CLAUDE.md 데모가 20분을 넘지 않도록 타이머 설정.
- Action Planning 5분은 반드시 확보한다. 이게 없으면 교육 실패.

### 리스크 대응

| 상황 | 대응 |
|---|---|
| 라이브 데모가 예상과 다른 결과 | 당황하지 않고 "지금 일어난 일을 함께 해석해봅시다" 프레이밍. 이것도 학습이다. |
| 네트워크 문제 | 녹화 백업 영상 재생 |
| 특정 참가자가 기술 토론으로 빠짐 | "좋은 질문입니다. 쉬는 시간에 이어서 얘기하시죠" |
| 반응이 전반적으로 미지근함 | L3 (미처리 액션 아이템 추출) 데모를 천천히 다시 보여준다. 가장 강력한 "아하" 포인트. |

## 세션 후 (당일~1주)

- [ ] Action Planning 워크시트 사진/스캔 수집
- [ ] 참가자에게 사이트 링크 + 설치 가이드 메일 발송
- [ ] 1주 후 "월요일 액션 어떻게 됐는지" 확인 메일 1통
- [ ] 강사 회고 기록 (무엇이 먹혔는가, 무엇을 바꿀 것인가)
