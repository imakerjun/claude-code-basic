# 2. Skills · 절차의 이름화 (5분)

> 출처: Anthropic 공식 문서 — [Skills](https://code.claude.com/docs/en/skills)

## 한 문장 정의

**[Skills](https://code.claude.com/docs/en/skills)는 반복되는 워크플로우에 이름을 붙여, 필요할 때 그 이름으로 부르는 장치다.**

## CLAUDE.md와의 차이

| | CLAUDE.md | Skills |
|---|---|---|
| 적용 시점 | 항상 | 필요할 때만 |
| 내용 | 원칙·제약 | 절차·순서 |
| 비유 | 사내 가이드 | 표준 운영 절차(SOP) |

## 실제 예시 — 이 교육 자료를 만들 때 사용한 Skill

**Skill 이름**: 전문가소환 (expert-panel)

**설명**: 주제에 대해 관련 학문 분야를 탐색하고, 근거 수준을 태그하며, 전문가 패널 토론을 거쳐 구현 설계까지 이어지는 워크플로우.

**호출**: 대화창에 "전문가 패널로 이 주제 토론해줘" 라고 입력.

**결과**: 매번 동일한 구조(학문 지도 → 근거 지형 → 패널 토론 → 구현 설계 → 실험 설계)로 산출물이 나옵니다. Skill 없이 매번 자연어로 지시하면 구조가 실행마다 달라집니다.

## 조직에서 Skill 후보 찾는 질문

> "매주/매월 반복되는 의사결정 절차가 무엇인가?"

### 후보 예시

- **고객 불만 분류** — 들어온 이슈를 5개 카테고리로 분류
- **채용 이력서 1차 스크리닝** — 공통 기준으로 평가
- **주간 경영 리포트 생성** — 지표 수집 + 코멘트
- **신규 기획안 리뷰** — 공통 프레임 점검

반복되는 절차는 모두 Skill 후보입니다.
