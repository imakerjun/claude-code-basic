# 1. CLAUDE.md · 규칙의 설계

> 출처: Anthropic 공식 문서 — [Manage Claude's memory](https://code.claude.com/docs/en/memory)

## 한 문장 정의

**[CLAUDE.md](https://code.claude.com/docs/en/memory)는 Claude Code가 매 세션 시작 시 먼저 읽어 이후 모든 대화에 자동 적용하는 규칙 파일이다.** 초안은 `/init`이 만들어 주고, 이후의 규칙은 **일하다가 마찰이 생긴 순간 Claude에게 말해서 파일에 반영**합니다. **무엇을 어떤 의도로 적느냐**가 하네스 설계(Böckeler의 *Guides*)의 시작점입니다.

## 라이브 데모 시나리오

핵심은 세 가지입니다 — **CLAUDE.md를 직접 쓰지 않는다**, **대화로 규칙을 합의한다**, **반영은 Claude가 한다**.

### 장면 1 · `/init`으로 초안 생성

빈 저장소에서 시작합니다.

```
/init
```

Claude가 저장소를 훑고 CLAUDE.md 초안을 만듭니다. 기술 스택, 디렉터리 구조, 빌드/테스트 명령 같은 *사실*은 AI가 스스로 채웁니다 — 사람이 손으로 옮겨 적을 자리가 아닙니다.

### 장면 2 · 초안 상태에서 실제 요청

규칙이 아직 없는 상태에서 요청합니다.

```
다음 주 회의 아젠다 초안을 3줄로 만들어줘. sample-team 근거 문서는 안 봐도 돼.
```

형식·톤이 일정하지 않은 일반적인 결과가 나옵니다. 이것이 "초안만 있고 가이드는 없는" 상태입니다.

### 장면 3 · 대화로 규칙을 합의

Claude에게 말합니다.

```
결과 톤이 들쭉날쭉해. 우리 팀은
존댓말로, 아젠다는 최대 5개, 각 항목 한 줄로 통일하고 싶어.
이 세 가지를 CLAUDE.md에 반영해줘.
```

Claude가 CLAUDE.md를 직접 Edit 해서 규칙을 추가합니다. 사람은 파일을 열지 않습니다.

### 장면 4 · 동일 요청 반복

같은 프롬프트를 다시 실행하면 결과가 달라집니다 — 존댓말, 5개 이하, 한 줄. 변경점은 방금 AI가 CLAUDE.md에 적은 세 줄뿐입니다.

> **이 데모의 숨은 메시지**: 하네스 설계는 "문서를 손으로 쓰는 일" 이 아닙니다. **일하다가 마찰을 느낀 순간, 그 마찰을 Claude에게 말해서 규칙으로 만드는 습관**입니다.

## 규칙 예시 (조직 도입 아이디어)

### 품질
- "한국어 문서는 `~합니다` 체를 쓴다"
- "숫자에는 반드시 출처를 명시한다"
- "한 줄 요약이 없는 제안은 수용하지 않는다"

### 보안·컴플라이언스
- "고객 데이터가 포함된 분석은 반드시 익명화부터 한다"
- "민감 필드(이메일·전화번호·주민번호)는 출력에 포함하지 않는다"
- "외부 전송 코드 작성 시 로그 기록을 필수로 한다"

### 프로세스
- "모든 SQL은 쓰기 전에 실행 계획을 확인한다"
- "커밋 메시지는 [JIRA-번호]로 시작한다"
- "PR 설명은 '왜'가 맨 위에 온다"
