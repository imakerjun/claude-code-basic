# 1. CLAUDE.md · 규칙의 설계

> 출처: Anthropic 공식 문서 — [Manage Claude's memory](https://code.claude.com/docs/en/memory)

## 한 문장 정의

**[CLAUDE.md](https://code.claude.com/docs/en/memory)는 이 저장소에서 Claude Code가 항상 따라야 할 규칙을 적어두는 파일이다.**

## 왜 중요한가

Claude Code는 매 세션마다 CLAUDE.md를 먼저 읽고, 이 파일에 적힌 원칙을 이후의 모든 대화에 자동 적용합니다. **한 파일을 편집하는 것이 조직의 AI 동작 기준을 바꾸는 것**과 동일한 효과를 냅니다.

## 라이브 데모 시나리오

### 장면 1 · 규칙 없는 상태

빈 CLAUDE.md 상태에서 요청합니다.

```
다음 주 임원회의 아젠다 초안을 3줄로 만들어줘. sample-team 근거 문서는 안 봐도 돼.
```

형식·톤이 일정하지 않은 일반적인 결과가 나옵니다.

### 장면 2 · 규칙 추가

`CLAUDE.md`를 열어 세 줄을 추가합니다.

```markdown
## 조직 규칙

- 모든 응답은 존댓말로 작성한다.
- 회의 아젠다는 최대 5개 항목까지만 작성한다.
- 각 아젠다 항목은 한 줄을 넘지 않는다.
```

### 장면 3 · 동일 요청 반복

같은 프롬프트를 다시 실행하면 결과가 달라집니다.

- 존댓말 유지
- 5개 이하 항목
- 각 항목 한 줄

변경점은 CLAUDE.md에 추가한 세 줄뿐입니다.

## 규칙 예시 (조직 도입 아이디어)

### 품질
- "한국어 문서는 `~합니다` 체를 쓴다"
- "숫자에는 반드시 출처를 명시한다"
- "한 줄 요약이 없는 제안은 수용하지 않는다"

### 보안·컴플라이언스
- "고객 데이터가 포함된 분석은 반드시 익명화부터 한다"
- "민감 필드(이메일·전화번호·주민번호)는 출력에 포함하지 않는다"
- "외부 전송 코드 작성 시 로그 기록을 필수로 한다"

### 프로세스
- "모든 SQL은 쓰기 전에 실행 계획을 확인한다"
- "커밋 메시지는 [JIRA-번호]로 시작한다"
- "PR 설명은 '왜'가 맨 위에 온다"
