# 3. Agents와 Hooks · 개념 소개

> 90분 입문 세션에서는 **개념만** 소개합니다.
>
> 출처: Anthropic 공식 문서 — [Subagents](https://code.claude.com/docs/en/sub-agents) · [Hooks](https://code.claude.com/docs/en/hooks)

## Agents · 역할 기반 위임

### 한 문장 정의

**[Agents](https://code.claude.com/docs/en/sub-agents)는 특정 역할을 맡은 전문 AI 일꾼이다. 각자 다른 CLAUDE.md와 Skills를 가질 수 있다.**

### 예시

- **코드 리뷰어 에이전트** — PR을 품질·보안 관점으로 검토
- **회의록 요약 에이전트** — 긴 회의록을 3줄로 요약
- **고객 이슈 분류 에이전트** — 문의를 카테고리로 분류

### 왜 나누는가

역할별로 필요한 성향이 다릅니다.

- "코드 리뷰어" 는 **보수적** — 의심스러우면 지적
- "아이디어 브레인스토머" 는 **관대** — 의심스러워도 제안

둘은 반대 성향이라 한 AI에게 둘 다 시키면 애매해집니다. Agents로 나누면 각각 명확해집니다.

## Hooks · 정책의 자동 집행

### 한 문장 정의

**[Hooks](https://code.claude.com/docs/en/hooks)는 특정 행동 전/후에 자동 실행되는 검사 장치다. "지켜주세요"가 아니라 "못 하게 막는다".**

### CLAUDE.md와의 차이

| | CLAUDE.md | Hooks |
|---|---|---|
| 성격 | 가이드라인 | 강제 집행 |
| 위반 시 | 모델이 잊을 수 있음 | **기계적으로 차단** |
| 용도 | 품질·톤 | 보안·컴플라이언스 |

### 예시

- **커밋 전 민감 정보 스캔** — API 키 포함 시 자동 차단
- **배포 전 테스트 통과 확인** — 실패 시 배포 불가
- **고객 데이터 접근 시 로그 기록** — 감사 추적

### 언제 Hooks가 필요한가

"어기면 사고가 나는 규칙"은 가이드라인(CLAUDE.md)으로 부족합니다. 강제력이 필요한 규칙은 Hooks로 집행합니다.
