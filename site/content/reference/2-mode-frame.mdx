# 2-모드 프레임 요약

이 교육이 사용하는 개념 프레임을 한 페이지로 정리합니다. 모두 1차 출처 기반입니다.

## 두 모드

| 모드 | 이름 | 한 줄 설명 | 1차 출처 |
|---|---|---|---|
| **1** | Vibe coding | 자연어 한 방 프롬프트로 만들어내는 즉흥 모드 | — |
| **2** | **Agentic coding + Harness engineering** | 모델의 자율 실행 루프와, 그 루프가 돌아가는 환경·규칙의 설계 | [Anthropic](https://code.claude.com/docs/en/best-practices) · [LangChain](https://blog.langchain.com/the-anatomy-of-an-agent-harness/) · [Fowler](https://martinfowler.com/articles/exploring-gen-ai/harness-engineering.html) |

## 모드 2 내부의 두 면

모드 2는 한 덩어리지만, 초점이 다른 두 면을 포함합니다.

| 구분 | Agentic coding | Harness engineering |
|---|---|---|
| 다루는 것 | 모델의 **행위 양식(루프)** | 루프가 도는 **환경·규칙** |
| 원문 정의 | *"Claude Code is an agentic coding environment… can read your files, run commands, make changes, and autonomously work through problems."* — Anthropic | *"Agent = Model + Harness."* — LangChain · **Guides/Sensors** 설계 — Böckeler |
| 초점 | **무엇을 하는가** | **어디서·어떤 규칙으로 하는가** |

> Anthropic의 Best Practices 문서는 agentic coding 환경을 정의한 직후 같은 문서에서 CLAUDE.md·permissions·hooks·subagents·skills 설정을 안내합니다. 즉 두 개념은 **한 세트**로 쓰이는 것이 정상입니다.

## 경계선

- **모드 1 → 모드 2 경계**: 모델이 스스로 파일을 열고·명령을 실행하며·자율적으로 문제를 풀어가는가
- **모드 2 내부의 경계 (선택)**: 루프의 *행위*에 주목하면 agentic, 그 루프가 도는 *환경의 설계*에 주목하면 harness

## 하네스를 설계할 시점인가

세 개 중 하나라도 "그렇다"가 나오면 하네스를 설계해야 할 시점입니다.

1. 여러 명이 Claude Code를 쓰고 있다
2. AI 생성물의 품질·톤이 들쭉날쭉하다
3. 보안·컴플라이언스 관점에서 "그러면 안 되는 일"이 눈에 들어온다

## 하네스의 네 재료

```
CLAUDE.md  →  원칙과 가이드      (사내 가이드 문서)
Skills     →  반복 절차의 이름   (SOP)
Agents     →  역할별 전문 일꾼   (팀의 전문 담당자)
Hooks      →  강제력 있는 검사   (게이트 키퍼)
```
