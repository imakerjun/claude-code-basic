# 3-tier 프레임 요약

세션의 핵심 개념을 한 페이지로 정리합니다.

## 세 층위

| Tier | 이름 | 누가 | 무엇을 | 한 줄 설명 |
|---|---|---|---|---|
| **T1** | Vibe coding | 누구나 | "이거 만들어줘" 한 번 | 즉흥적 한 방 프롬프트 |
| **T2** | Agentic coding | 실무자 | 모델이 파일·명령·수정을 자율 실행 | 행위 양식(루프) |
| **T3** | **Harness engineering** | **팀으로 AI를 쓰려는 사람** | CLAUDE.md·Skills·Agents·Hooks | 루프가 도는 환경·규칙 설계 |

## 경계선

- **T1 → T2 경계**: "모델이 스스로 계획·실행·검증을 반복하는가"
- **T2 → T3 경계**: "그 루프가 도는 환경·도구·규칙을 사람이 설계하는가"

## 출처 기반 정의

- **Agentic coding** — Anthropic, [Claude Code Best Practices](https://code.claude.com/docs/en/best-practices): *"Claude Code is an agentic coding environment… can read your files, run commands, make changes, and autonomously work through problems."* → 모델의 **행위 양식**.
- **Harness engineering** — LangChain, [The Anatomy of an Agent Harness](https://blog.langchain.com/the-anatomy-of-an-agent-harness/): **Agent = Model + Harness** (모델 외 전부: 도구·컨텍스트·가드레일·피드백·관측). Martin Fowler, [Harness Engineering](https://martinfowler.com/articles/exploring-gen-ai/harness-engineering.html): **Guides(피드포워드)** 와 **Sensors(피드백)** 설계 실천. → 루프가 도는 **환경·규칙**.

## T3를 설계할 시점인가

세 개 중 하나에라도 "그렇다" 가 나오면 T3를 설계해야 할 시점입니다.

1. 우리 팀 여러 명이 Claude Code를 쓰고 있다
2. AI 생성물의 품질·톤이 들쭉날쭉하다
3. 보안·컴플라이언스 관점에서 "그러면 안 되는 일" 이 눈에 들어온다

## 하네스의 네 재료

```
CLAUDE.md  →  원칙과 가이드      (사내 가이드 문서)
Skills     →  반복 절차의 이름   (SOP)
Agents     →  역할별 전문 일꾼   (팀의 전문 담당자)
Hooks      →  강제력 있는 검사   (게이트 키퍼)
```
