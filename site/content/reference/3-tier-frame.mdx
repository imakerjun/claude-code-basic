# 3-tier 프레임 요약

세션의 핵심 개념을 한 페이지로 정리합니다.

## 세 층위

| Tier | 이름 | 누가 | 무엇을 | 한 줄 설명 |
|---|---|---|---|---|
| **T1** | Vibe coding | 아무나 | "이거 만들어줘" 한 번 | 즉흥적 한 방 프롬프트 |
| **T2** | Agentic coding | 실무자 | Claude가 계획·실행·검증 | 자기 수정 루프 |
| **T3** | **Harness engineering** | **의사결정자·리더** | CLAUDE.md·Skills·Agents·Hooks | 조직이 AI와 일하는 방식의 설계 |

## 임원 관점의 한 문장

> **T1은 개인이 씁니다.**
> **T2는 실무자가 씁니다.**
> **T3는 — 여러분이 합니다.**

## 경계선

- **T1 → T2 경계**: "계획을 스스로 세우는가"
- **T2 → T3 경계**: "조직 단위로 기준이 적용되는가"

## 임원의 판단 질문

세 개 중 하나에라도 "그렇다" 가 나오면 T3를 설계해야 할 시점입니다.

1. 우리 팀 여러 명이 Claude Code를 쓰고 있다
2. AI 생성물의 품질·톤이 들쭉날쭉하다
3. 보안·컴플라이언스 관점에서 "그러면 안 되는 일" 이 눈에 들어온다

## 하네스의 네 재료

```
CLAUDE.md  →  원칙과 가이드      (사내 가이드 문서)
Skills     →  반복 절차의 이름   (SOP)
Agents     →  역할별 전문 일꾼   (팀의 전문 담당자)
Hooks      →  강제력 있는 검사   (게이트 키퍼)
```

## 오늘 월요일 오전, 나의 한 줄

네 재료 중 **CLAUDE.md 한 줄부터** 시작합니다. 나머지는 CLAUDE.md가 자리 잡은 후 자연스럽게 따라옵니다.
