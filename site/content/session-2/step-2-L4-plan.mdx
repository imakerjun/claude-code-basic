# 2. L4 · 계획 수립

## 프롬프트

```
다음 주 임원회의 아젠다를 만들려고 해.
sample-team/ 문서들을 근거로 아젠다 계획을 먼저 세워줘.
- 우선순위가 높은 항목부터
- 각 항목마다 근거 문서를 링크로 달아줘
- 아직 파일을 만들지는 말고, 계획만 보여줘
```

## 핵심

"아직 파일을 만들지는 말고" 한 줄이 Claude의 행동을 **실행**에서 **기획**으로 전환시킵니다. L1~L3이 "물어보고 답을 받는" 관계였다면, L4부터 Claude가 **작업을 쪼개고, 순서를 정하고, 스스로 실행**합니다. 이것이 에이전틱 코딩의 전형적 모습입니다.

계획의 **기준**은 여전히 사람이 정합니다 → 이 기준을 고정하는 것이 세션 3 · 하네스 엔지니어링입니다.
