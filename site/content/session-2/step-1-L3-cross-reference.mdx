# 1. L3 · 여러 문서 가로지르기

## 프롬프트

```
sample-team/회의록/ 폴더의 회의록 3개를 모두 읽고,
아직 처리되지 않은 액션 아이템만 담당자별로 정리해줘.
완료된 것은 제외해줘.
```

## 왜 이 프롬프트인가

다음 세 가지 작업을 한 번에 요구합니다.

- 여러 회의록을 가로지르기
- 완료/미완료 구분
- 담당자별 재분류

개별 작업은 단순하지만, 세 가지를 사람이 수작업으로 처리할 때와 Claude Code가 처리할 때의 소요 시간 차이를 직접 체감하는 것이 포인트입니다.

## 예상 출력

```
미처리 액션 아이템 (담당자별)

이대리
- [2026-03-29] 본부 대시보드 KR1/KR2 지표 스키마 협의 (4/3까지)

최매니저
- [2026-03-29] severity 2%/5분/shadow 반영한 정의서 v2 배포 (4/5까지)
- [2026-03-29] 선물하기 쿠폰 발급 실패 원인 확인 (4/5까지)

정개발
- [2026-03-29] merchant-onboarding 레거시 파악 + 현황 리포트 (4/5까지)
- [2026-03-29] KR2 알림 파이프라인 아키텍처 초안 (4/12까지)

박주임
- [2026-03-29] 4/5~4/6 KCP 정기 점검 영향 리포트 (4/3까지)
```

## 핵심 개념

단일 파일 요약(L1)·폴더 탐색(L2)을 넘어, **여러 문서의 상태를 교차 판단**하는 단계입니다. 이때부터 "기존 대화형 AI와의 차이"가 눈에 띄게 벌어집니다.
