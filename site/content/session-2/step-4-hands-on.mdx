# 4. 실습 (15분)

## 규칙

- 각자 노트북에서 Claude Code 실행
- 작업 디렉터리: `claude-code-basic/` 또는 `sample-team/`
- 성공 보장 프롬프트를 사용합니다 (아래)
- 실패해도 괜찮습니다. 옆 사람에게 물어봐도 됩니다.

## 성공 보장 프롬프트

```
sample-team 폴더 전체를 훑어보고 이 팀의 Q2 주요 과제 3개를 우선순위와 함께 알려줘.
```

## 확장 프롬프트 (여유 있는 분용)

한 단계 더 해보고 싶으시다면 아래 중 하나를 시도해보세요.

```
최근 회의록에서 '결제 오류 severity' 관련된 논의 맥락을 시간 순으로 정리해줘.
```

```
Q2 OKR 달성에 가장 큰 리스크 3개를 근거 문서와 함께 알려줘.
```

```
신규 입사자 정개발님이 첫 주에 읽어야 할 문서 우선순위를 정해줘.
```

## 실습 직후 30초 페어 회고

옆 사람과 한 번씩 이 질문에 답합니다. 답변은 강요하지 않습니다.

> **"방금 무엇이 놀라웠나요?"**

이 30초가 경험을 기억으로 고정시키는 가장 중요한 순간입니다.

## 강사 역할

- 실습 중 **개입하지 않습니다**. 본인 손으로 성공 경험하는 것이 목적입니다.
- 질문 받으면 "좋은 질문입니다. 잠시 후 T3에서 답이 될 수도 있습니다" 라고 일단 넘깁니다.
- 도저히 안 되는 분만 개별 지원합니다.
- 실습 끝날 때 "방금 놀라운 결과가 나온 분?" 한두 명에게 발표 요청합니다. **이게 다른 임원들에게 가장 강력한 증거** 입니다.
