# Claude Code Basic

**90분 Claude Code 입문 워크숍**

## 목적과 목표

Claude Code로 바이브 코딩을 넘어 **에이전틱 코딩·하네스 엔지니어링으로 가는 첫 걸음**을 직접 체험해 봅니다.

> 이 워크숍은 **웹앱·코드를 만드는 자리가 아닙니다.** Claude Code가 어떻게 움직이는지를 체감하고, 그 움직임을 어떻게 설계할지에 대한 어휘와 첫 경험을 가지고 돌아가는 자리입니다.

## 90분 끝에 무엇이 남는가

세 층위의 산출물이 남습니다. "웹앱"이 아니라 **agentic 루프와 하네스 설계의 첫 흔적**입니다.

### 1. 손에 남는 것 — 파일로 저장되는 산출물

| 산출물 | 어디서 나오나 | 의미 |
|---|---|---|
| `sample-team/회의록/2026-04-12-회의-아젠다.md` | 세션 2 · L5 데모 | 한 줄 지시로 Claude가 다문서를 가로질러 **직접 생성한 회의 아젠다 파일**. agentic 루프가 만든 첫 산출물 |
| 본인 노트북의 Claude Code 대화 기록 | 세션 2 · 실습 | 본인 손으로 돌린 첫 agentic 루프의 기록. 세션 후에도 열어볼 수 있음 |
| 규칙이 추가된 `CLAUDE.md` | 세션 3 · 하네스 데모 | "응답은 존댓말로", "아젠다는 5개 이하"처럼 **한 줄 규칙으로 AI 동작을 바꾼** 첫 하네스 설계 예시 |

### 2. 눈에 남는 것 — 직접 본 장면

- **L2 장면**: "어느 파일을 열지 지시하지 않았는데 Claude가 Glob·Read를 스스로 호출해 폴더 전체를 훑는" agentic 루프의 첫 증거
- **L5 장면**: 한 줄 자연어 지시가 → 문서 탐색 → 우선순위 판단 → 계획 수립 → 파일 생성 4단계로 확장되는 전형적 agentic 실행 루프
- **CLAUDE.md 장면**: 같은 요청을 한 번은 빈 상태에서, 한 번은 세 줄 규칙을 추가한 상태에서 실행했을 때 결과가 달라지는 순간 — 하네스가 실제로 "동작하는" 장면

### 3. 머리에 남는 것 — 설계 어휘

- **2-모드 프레임**: Vibe coding / Agentic coding + Harness engineering — 1차 출처(Anthropic · LangChain · Martin Fowler) 기반
- **하네스의 네 가지 재료**: CLAUDE.md · Skills · Agents · Hooks
- **판단 기준 한 줄**: *Agent = Model + Harness* — 모델을 바꿀 수 없어도, 하네스는 설계할 수 있다

## 2-모드 프레임 (출처 기반)

Claude Code를 설명하는 두 가지 층위입니다. 이 교육 전체는 이 구분을 축으로 진행됩니다.

### 1. Vibe coding — 즉흥 한 방 모드

자연어 한 번으로 무언가를 만들어내는 모드입니다. 체험은 강렬하지만 **일회성**이고, 조직이 손댈 레버가 거의 없습니다.

### 2. Agentic coding + Harness engineering — 설계 가능한 층위

한 덩어리로 묶입니다. **에이전틱 코딩**은 모델의 *행위 양식(자율 실행 루프)* 이고, **하네스 엔지니어링**은 *그 루프가 돌아가는 환경·도구·규칙의 설계* 입니다. 둘은 대립이 아니라 층위 관계입니다.

| 구분 | Agentic coding | Harness engineering |
|---|---|---|
| 다루는 것 | 모델의 **행위 양식(루프)** | 루프가 도는 **환경·규칙** |
| 한 문장 | 파일을 읽고, 명령을 실행하고, 수정하고, 자율적으로 문제를 풀어간다 | Agent = Model + **Harness** (도구·컨텍스트·가드레일·피드백·관측) |
| 1차 출처 | [Anthropic — Claude Code Best Practices](https://code.claude.com/docs/en/best-practices) | [LangChain — Anatomy of an Agent Harness](https://blog.langchain.com/the-anatomy-of-an-agent-harness/) · [Martin Fowler — Harness Engineering](https://martinfowler.com/articles/exploring-gen-ai/harness-engineering.html) |
| 원문 인용 | *"Claude Code is an agentic coding environment… can read your files, run commands, make changes, and autonomously work through problems."* | *"Agent = Model + Harness."* · Böckeler — **Guides(피드포워드)** 와 **Sensors(피드백)** 설계 실천 |

> Anthropic의 Best Practices 문서 자체가 이 구조를 따릅니다 — "agentic coding environment"를 정의한 뒤, 같은 문서에서 CLAUDE.md·permissions·hooks·subagents·skills 같은 하네스 구성요소를 설정하라고 안내합니다.

## 어떤 경험을 하게 되나요

90분 동안 위 두 모드를 차례로 체감합니다.

### 1. "어, 이거 챗봇이랑 다르네" — 에이전틱 루프의 첫 체감

한 줄 자연어로 Claude Code가 저장소 안 여러 파일을 스스로 열어 읽고, 요약하는 모습을 봅니다. 파일을 어느 것 열지 사람이 지시하지 않았다는 사실이 눈에 띄는 순간이 포인트입니다.

### 2. "계획을 스스로 세운다고?" — 루프의 심화

한 번의 요청으로 Claude Code가 여러 문서를 가로질러 판단하고, 작업을 쪼개고, 순서대로 실행해 파일까지 만들어내는 과정을 함께 봅니다. 중간에 본인 노트북으로 같은 흐름을 짧게 따라 해봅니다.

### 3. "그 루프가 어떤 규칙으로 돌 것인가" — 하네스 설계

**CLAUDE.md, Skills, Agents, Hooks** 네 가지 재료로 "Claude Code가 어떤 규칙 안에서 돌지"를 직접 설계합니다. 한 줄짜리 규칙을 추가하면 다음 응답이 즉시 달라지는 장면을 라이브로 봅니다.

## 이 워크숍이 다루지 않는 것

- 코드 문법·프로그래밍 기초 — 이 워크숍은 코드를 짜는 자리가 아닙니다
- 모델 내부 원리, LLM 학습 과정 — 의사결정에 필요한 만큼만 다룹니다
- "어떤 도구가 가장 좋은가" 비교 — Claude Code 한 가지로 개념을 끝까지 따라갑니다

## 90분 흐름

| 구간 | 섹션 | 모드 |
|---|---|---|
| 0~10 | 오프닝 · 2-모드 프레임 소개 | 전체 |
| 10~25 | 바이브 대비 + 에이전틱 루프 첫 체감 (L1~L2) | Vibe → Agentic |

| 55~75 | **하네스 엔지니어링** — CLAUDE.md / Skills / Agents / Hooks | Harness |
| 75~90 | 도입 시 고려사항 | 조직 |
