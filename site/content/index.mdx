# Claude Code Basic

**우아한형제들 임원진 대상 90분 Claude Code 입문 워크숍**

## 한 줄 소개

Claude Code를 "코드 짜는 도구"로 배우는 자리가 아니라, **조직이 AI와 어떻게 일할지를 설계하는 자리**입니다.

## 어떤 경험을 하게 되나요

90분 동안 세 단계의 사고 전환을 차례로 거칩니다.

### 1. "어, 이거 ChatGPT랑 다르네"

강사가 직접 노트북을 띄우고, 한 줄 자연어로 Claude Code가 저장소 안 여러 파일을 스스로 열어 읽고, 표를 만들고, 요약하는 모습을 보여드립니다. 이 구간의 목표는 감탄이 아니라 **"대화형 AI와 에이전트형 AI는 다른 종류구나"** 라는 인식이 자리 잡는 것입니다.

### 2. "계획을 스스로 세운다고?"

한 번의 질문에 Claude Code가 작업을 쪼개고, 순서를 정하고, 파일을 읽고 쓰며 스스로 검증까지 도는 과정을 함께 봅니다. 중간에 본인 노트북으로 같은 흐름을 짧게 따라 해보는 시간이 있습니다 (실습이 부담되시면 옆자리와 페어로 보셔도 됩니다). 이 구간의 목표는 **"AI가 일하는 방식이 단답형에서 루프형으로 바뀌고 있다"** 를 손으로 느끼는 것입니다.

### 3. "그러면 우리 조직은 무엇을 설계해야 하지?"

이 워크숍의 본론입니다. **CLAUDE.md, Skills, Agents, Hooks** 라는 네 가지 재료로 "AI가 우리 조직에서 어떻게 행동해야 하는지" 를 직접 설계합니다. 한 줄짜리 규칙을 추가하면 Claude Code의 다음 응답이 즉시 달라지는 장면을 라이브로 보여드리고, 마지막에는 **본인 조직에 들일 한 줄** 을 워크시트로 남기고 마칩니다.

## 무엇을 가지고 돌아가시는가

- **언어**: Vibe / Agentic / Harness 라는 3-tier 프레임으로 AI 도입 단계를 구분해 말할 수 있게 됩니다
- **판단 기준**: 우리 팀이 지금 어느 단계에 있는지, 다음 단계로 가려면 무엇을 설계해야 하는지 스스로 판단하는 기준이 생깁니다
- **첫 한 줄**: 월요일 오전에 본인 조직 CLAUDE.md에 추가할 **단 한 줄** 을 직접 써서 가져가십니다

## 이 워크숍이 다루지 않는 것

- 코드 문법과 프로그래밍 기초 — 임원진은 코드를 짤 필요가 없습니다
- 모델 내부 원리, LLM 학습 과정 — 의사결정에 필요한 만큼만 다룹니다
- "어떤 도구가 가장 좋은가" 비교 — Claude Code 한 가지로 개념을 끝까지 따라갑니다

## 90분 흐름

| 구간 | 섹션 | Tier |
|---|---|---|
| 0~10 | 오프닝 · 3-tier 프레임 소개 | 전체 |
| 10~25 | T1 체험 — 문서 분석 L1~L2 | T1 |
| 25~40 | T2 이해 — 문서 분석 L3~L5 | T2 |
| 40~55 | 실습 — 본인 노트북에서 따라 하기 | T1→T2 |
| 55~75 | **T3 하네스 엔지니어링** — CLAUDE.md / Skills / Agents / Hooks | **T3** |
| 75~90 | 도입 시 고려사항 + 월요일의 한 줄 | 조직 |
