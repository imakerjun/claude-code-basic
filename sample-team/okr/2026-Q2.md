# 2026 Q2 팀 OKR — 결제플랫폼팀

- 소속: 결제서비스본부 / 결제플랫폼팀
- 팀장: 김팀장 (보고선: 결제서비스본부장)
- 작성: 2026-03-24, 본부장 승인 2026-03-27
- 연동 문서: `회의록/2026-03-15`, `회의록/2026-03-22`, `회의록/2026-03-29`

## 배경

- 배민1 단건배달 전환 확대로 2026년 1월 기준 결제 TPS가 전년 동월 대비 +38%. 피크 시간대(18:00~20:00) 평균 1,240 TPS.
- 2/27 KCP 간편결제 스펙 변경 대응으로 Q1 KR "온보딩 자동화"가 2주 지연되어 달성률 62%에 그침.
- 사장님광장 신규 가맹점 온보딩은 현재도 CS운영팀이 엑셀 기반 수기 처리 중이며, 주말 유입분은 월요일에 몰려 SLA(영업일 2일) 초과 사례가 3월에만 17건 발생.

**Objective**: 결제 플랫폼의 신뢰성과 확장성을 한 단계 끌어올려, 배민1 단건배달 확장 국면에서 "결제가 병목이 되지 않는 팀"이 된다.

## KR1. 사장님광장 신규 가맹점 온보딩 자동화 — 월 12건(수동) → 월 80건(자동화율 90%)

- 담당: 이대리(PM), 정개발(백엔드), 박주임(파트너사 API 검증)
- 현재: 월 평균 12건, 1건당 CS운영팀 평균 처리 시간 3시간 40분 (사업자 진위확인 → PG 계약 연동 → 정산 계좌 검증 → 메뉴 시딩)
- 목표:
  - Q2 말 기준 월 80건 처리, 이 중 90% 이상이 CS 개입 없이 완료
  - 1건당 평균 처리 시간 30분 이내 (사람 개입 구간은 KYC 예외 케이스만)
- 측정:
  - `onboarding-metrics` 대시보드 (Grafana) 월말 스냅샷
  - CS운영팀 티켓 시스템의 "가맹점 온보딩" 카테고리 주간 리포트
- 리스크:
  - 국세청 사업자 진위확인 API의 일일 호출 한도(10,000건) — 대량 이벤트 시 병목 가능
  - 정산 계좌 검증을 위해 은행별 1원 인증 응답 지연(평균 18초, p99 2분)

## KR2. 배민페이 결제 이상 인지 시간 평균 28분 → 5분 이내

- 담당: 최매니저(운영), 정개발(알림 파이프라인)
- 현재:
  - Q1 평균 인지 시간 28분 (수동 대시보드 조회 기반)
  - 2/27 KCP 장애 당시 최초 인지까지 41분 소요, 누적 피해 결제 약 7,200건
- 목표:
  - 결제 실패율·승인 지연·PG별 타임아웃을 1분 단위 윈도우로 감시
  - 이상 탐지 후 5분 이내 Slack `#payment-alerts` 및 온콜 페이저(PagerDuty)로 자동 통보
  - severity P0/P1/P2 기준 정착 (→ `회의록/2026-03-29` 미해결 이슈 연결)
- 측정: "이상 발생 시각" − "최초 대응 코멘트 시각" (PagerDuty 로그 기반) 월간 중앙값
- 리스크: 오탐(false positive)로 온콜 피로 누적. 초기 2주는 shadow 모드 운영 필요.

## KR3. PG 3사(토스페이먼츠·KCP·나이스페이) 호환성 리스크 가시화

- 담당: 박주임
- 현재: 파트너사 API 변경 공지는 이메일·팩스로 산발 수신, 영향 분석은 건별 수작업. 2/27 사건도 공지 메일이 스팸함에 있었음.
- 목표:
  - 3사 공지·문서 변경 이력을 주 1회 자동 수집하여 월간 "PG 호환성 리스크 리포트" 발행
  - Q2 말까지 총 3회 발행, 각 리포트에 영향 범위·마이그레이션 기한·담당자 포함
- 측정: Q2 말 시점 발행 횟수 및 본부장 리뷰 완료 건수
- 리스크: PG사별 공지 채널이 표준화되어 있지 않아 수집 누락 가능성
